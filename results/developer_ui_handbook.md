# СПРАВОЧНИК РАЗРАБОТЧИКА ИНТЕРФЕЙСА (DEEPUTIN Forensic SCAP v2.0)

Этот файл содержит подробную спецификацию структуры данных, файлов и метрик, полученных в результате работы Sequential Calibration-Aware Pipeline (SCAP v2.0). Эти файлы оптимизированы для быстрой загрузки, фильтрации и визуализации в интерфейсе.

Все файлы лежат в корневой папке результатов: `/Volumes/SDCARD/storage/`

---

## 📂 Обзор скомпилированных файлов данных (Root-уровень)

Для исключения необходимости сканирования тысяч папок все данные собраны в 4 плоские таблицы (CSV) и 1 общий JSON-файл в корне директории хранения:

### 1. `master_photos.csv` — Справочник всех фотографий
Содержит метаданные и ракурсные параметры для каждой фотографии (как основной, так и калибровочной).
* **Поля:**
  * `photo_id` (string): Уникальный идентификатор фото (например, `2014_08_27_y-4p7r0`).
  * `dataset_type` (string): Тип датасета — `main` (основная хронология) или `calibration` (калибровочное фото).
  * `date` (ISO Date): Точная дата съемки (парсится из названия).
  * `yaw` (float): Угол поворота головы влево/вправо в градусах.
  * `pitch` (float): Угол наклона головы вверх/вниз в градусах.
  * `roll` (float): Угол крена головы в градусах.
  * `bucket` (string): Ракурсная группа (например, `frontal`, `left_profile`, `right_threequarter_mid`).
  * `expression_jaw_open` (boolean): Флаг приоткрытого рта (исключает нестабильные зоны).
  * `expression_smile` (boolean): Флаг улыбки.
  * `path` (string): Полный путь к исходному файлу изображения.

### 2. `master_pairs.csv` — Сводка хронологических переходов (пар)
Связывает фотографии в хронологические цепочки внутри ракурсных групп и содержит параметры калибровочного сопоставления.
* **Поля:**
  * `pair_id` (string): Идентификатор перехода (формат: `photoA__photoB`).
  * `group` (string): Ракурсная группа перехода.
  * `photo_A` (string): Идентификатор предыдущего фото.
  * `photo_B` (string): Идентификатор последующего фото.
  * `photo_B_date` (ISO Date): Дата последующего фото (для временной шкалы).
  * `calibration_quality` (string): Качество калибровки шума позы (`high`, `medium`, `marginal`, `low`). При качестве `low` рекомендуется предупреждать пользователя.
  * `calibration_A` (string): Калибровочное фото, использованное для канонизации фото A.
  * `calibration_B` (string): Калибровочное фото, использованное для канонизации фото B.
  * `pose_distance_A` (float): Евклидово расстояние позы фото А от его калибровочной пары (чем меньше, тем точнее калибровка).
  * `pose_distance_B` (float): Евклидово расстояние позы фото B от его калибровочной пары.
  * `approximate_match` (boolean): Было ли сопоставление приблизительным (если точной калибровочной пары по позе не нашлось).
  * `anomalies_count` (integer): Количество анатомических аномалий, найденных в этом переходе.

### 3. `master_metrics.csv` — Все анатомические метрики (Длинный формат)
Полная база вычисленных пропорций лица для всех пар. Самый ценный файл для построения интерактивных графиков.
* **Поля:**
  * `pair_id` (string): Идентификатор хронологического перехода.
  * `group` (string): Ракурсная группа.
  * `photo_A` (string): Идентификатор первого фото.
  * `photo_B` (string): Идентификатор второго фото.
  * `metric_name` (string): Имя биометрического параметра (см. список метрик ниже).
  * `raw_value` (float): Исходная дельта (разность) метрики между А и В до калибровки.
  * `corrected_value` (float): Итоговая дельта метрики после вычитания систематического шума позы (главный показатель для анализа).

### 4. `master_anomalies.csv` — Обнаруженные анатомические аномалии
Плоский список всех детекций, где очищенное значение дельты метрики превысило статистический порог стабильности.
* **Поля:**
  * `pair_id` (string): Идентификатор хронологического перехода.
  * `group` (string): Ракурсная группа.
  * `photo_A` (string): Идентификатор первого фото.
  * `photo_B` (string): Идентификатор второго фото.
  * `metric_name` (string): Изменившийся параметр лица.
  * `value` (float): Величина девиации (отклонения).
  * `threshold` (float): Статистический порог, при превышении которого фиксируется аномалия.
  * `severity` (string): Степень критичности аномалии (`warn` — легкое отклонение, `danger` — критическая аномалия/скачок пропорций).

### 5. `master_ui_data.json` — Единый Payload для Frontend-приложений
Если ваше приложение построено на React/Vue/Svelte/HTML5, вы можете загрузить один этот JSON-файл. Он содержит всю сводную информацию, плоские массивы фотографий, пар и аномалий в единой древовидной структуре.

## 📊 Список биометрических метрик (metric_name)

Алгоритм извлекает и сравнивает следующие ключевые зоны лица:

1. **`eyebrow_distance`** (Межбровное расстояние) — Изменение расстояния между внутренними точками бровей.
2. **`eyebrow_height_left` / `eyebrow_height_right`** (Высота бровей) — Положение бровей относительно глаз.
3. **`eye_aperture_left` / `eye_aperture_right`** (Степень раскрытия глаз) — Вертикальный размер глазной щели.
4. **`nose_width`** (Ширина крыльев носа) — Горизонтальный размер носа.
5. **`nose_bridge_width`** (Ширина переносицы).
6. **`nose_length`** (Длина носа от переносицы до кончика).
7. **`lip_thickness_upper` / `lip_thickness_lower`** (Толщина губ) — Индивидуальная высота верхней и нижней губы.
8. **`mouth_width`** (Ширина рта).
9. **`jaw_width`** (Ширина челюсти) — Угловое расстояние нижней челюсти.
10. **`forehead_height`** (Высота лба) — Расстояние от бровей до линии роста волос.
11. **`cheekbone_width`** (Ширина скул).

## 🎨 Рекомендации по построению интерфейса (UI/UX)

Для максимальной информативности рекомендуется реализовать 4 интерактивных экрана:

### Экран 1: Главная панель (Dashboard Overview)
* **Показатели:** Общее количество фотографий (1947), количество переходов (1723), количество найденных аномалий (15120).
* **Интерактивный график:** Круговая диаграмма распределения ракурсов по группам (на основе `bucket` в `master_photos.csv`).
* **Качество данных:** Индикатор распределения качества калибровки (`high`/`medium`/`marginal`/`low` из `master_pairs.csv`).

### Экран 2: Хронологическая шкала (Chronology Timeline)
* **Компонент:** Интерактивная таймлайн-линия (рекомендуется библиотека Vis.js или D3.js).
* **Данные:** На оси X располагаются точки по `photo_B_date`. 
* **Логика переходов:** При клике на отрезок времени между точками А и В загружается карточка пары из `master_pairs.csv`. В ней выводятся:
  * Фото А и Фото В (рядом для визуального сравнения экспертом).
  * Использованные калибровочные пары (`calibration_A`, `calibration_B`).
  * Список зафиксированных аномалий для этого конкретного шага из `master_anomalies.csv`.

### Экран 3: Графики динамики пропорций (Biometric Charts)
* **Компонент:** Линейный график динамики изменения конкретного параметра лица во времени (например, `lip_thickness_upper`).
* **Данные:** Загружаются из `master_metrics.csv` для выбранной группы ракурса (например, `frontal`).
* **Двойная линия:**
  * **Красная пунктирная линия** — `raw_value` (сырые геометрические изменения, содержащие шум позы).
  * **Зеленая сплошная линия** — `corrected_value` (очищенные от ракурсного шума анатомические изменения).
* **Инсайт:** В местах резких пиков зеленой линии (скачков пропорций) выводятся маркеры аномалий.

### Экран 4: Матрица стабильности (Anomaly Heatmap)
* **Компонент:** Тепловая карта (Heatmap) или таблица-сетка.
* **Ось Y:** Биометрические параметры (eyebrow_distance, nose_width и т.д.).
* **Ось X:** Хронологическая шкала времени.
* **Отображение:** Ячейка загорается желтым (`warn`) или красным (`danger`) цветом в те моменты времени, когда дельта метрики вышла за рамки нормы. Это позволяет эксперту за секунду увидеть, в какие периоды происходили резкие изменения анатомии лица.

## 🚀 Советы по оптимизации производительности
1. **Пагинация таблиц:** Файл `master_metrics.csv` содержит более 41 тыс. строк. Загружайте и отрисовывайте его на графиках лениво (lazy-load), отфильтровав по выбранному параметру лица (`metric_name`) и ракурсу (`group`).
2. **Кэширование изображений:** Фотографии лежат на внешней карте. Для ускорения отображения во фронтенде рекомендуется настроить кэширование или превью-генерацию изображений в уменьшенном разрешении.
