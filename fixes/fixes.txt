### 🔄 ЭТАП 1.2: ЗАМЕНА ШУМНЫХ МЕТРИК НА 3D-ИНВАРИАНТЫ

Здесь мы реализуем твое требование: **не удалять зоны, а менять математику**. Мы заменим 4 метрики с CV > 15-500% на их строгие трехмерные аналоги в `scoring.py`.

#### ЗАМЕНА 1: `canthal_tilt` ➔ `canthal_tilt_3d_coronal`
**Почему шумело:** Считалось через `atan2(dy, dx)` в мировых координатах. При повороте головы (yaw) `dx` стремится к нулю из-за перспективы, вызывая деление на микрочисло и шум в 2000%.
**Новая логика:** Проецируем вектор глаза на строго корональную плоскость лица (перпендикулярную `face_normal`), полностью нивелируя влияние поворота головы.

*Новый код (добавить в `scoring.py`):*
```python
def calc_tilt_3d_coronal(p_inner, p_outer, face_normal):
    """
    Вычисляет наклон глазной щели строго в корональной плоскости лица.
    Устойчиво к yaw-вращениям до 70 градусов.
    """
    # Вектор от внутреннего угла к внешнему
    eye_vector = p_outer - p_inner
    
    # Проецируем вектор на плоскость лица (удаляем Z-компоненту относительно нормали лица)
    eye_vector_proj = eye_vector - np.dot(eye_vector, face_normal) * face_normal
    
    # Нормализуем
    eye_vector_proj = eye_vector_proj / (np.linalg.norm(eye_vector_proj) + 1e-8)
    
    # Определяем горизонтальную ось лица (cross product нормали и вектора "вверх" Y=[0,-1,0])
    up_vector = np.array([0, -1, 0]) 
    face_horizontal = np.cross(face_normal, up_vector)
    face_horizontal = face_horizontal / (np.linalg.norm(face_horizontal) + 1e-8)
    
    # Считаем угол между спроецированным вектором глаза и горизонталью лица
    sin_theta = np.linalg.norm(np.cross(face_horizontal, eye_vector_proj))
    cos_theta = np.dot(face_horizontal, eye_vector_proj)
    
    return math.degrees(math.atan2(sin_theta, cos_theta))

# В extract_macro_bone_metrics:
# БЫЛО: metrics["canthal_tilt_L"] = calc_tilt(...)
# СТАЛО:
metrics["canthal_tilt_3d_L"] = calc_tilt_3d_coronal(canthus_L_inner, canthus_L_outer, face_plane_normal)
metrics["canthal_tilt_3d_R"] = calc_tilt_3d_coronal(canthus_R_inner, canthus_R_outer, face_plane_normal)
```

#### ЗАМЕНА 2: `chin_offset_asymmetry` ➔ `gnathion_midline_deviation`
**Почему шумело:** Подбородок брался как среднее углов челюсти (gonions), а асимметрия считалась от абстрактного центра, что делало ее зависимой от roll и yaw.
**Новая логика:** Строим строгую анатомическую среднюю линию (Midline) через Nasion (переносица) и Subnasale (основание носа). Считаем кратчайшее 3D-расстояние (перпендикуляр) от реальной точки подбородка (Gnathion) до этой линии.

*Новый код (добавить в `scoring.py`):*
```python
def calc_point_to_line_distance(point, line_p1, line_p2):
    """Кратчайшее 3D расстояние от точки до прямой, заданной двумя точками."""
    line_vec = line_p2 - line_p1
    point_vec = point - line_p1
    cross_prod = np.cross(line_vec, point_vec)
    return np.linalg.norm(cross_prod) / (np.linalg.norm(line_vec) + 1e-8)

# В extract_macro_bone_metrics (нужно добавить Nasion и Subnasale в MACRO_BONE_INDICES):

subnasale_pt = np.mean(vertices[_idx('subnasale')], axis=0)
gnathion_pt = np.mean(vertices[_idx('chin_center')], axis=0) # Реальный подбородок

# Абсолютное смещение подбородка от анатомической центральной линии
metrics["gnathion_midline_deviation_ratio"] = calc_point_to_line_distance(
    gnathion_pt, nasion_pt, subnasale_pt
) / zygomatic_breadth
```

#### ЗАМЕНА 3: `orbital_asymmetry_index` ➔ `orbital_3d_perimeter_ratio`
**Почему шумело:** Считалась разница bounding box-ов глазниц. При разном yaw проекция ближнего глаза больше дальнего.
**Новая логика:** Считаем 3D-периметр (сумму длин отрезков) по контуру глазницы. Длина кривой в 3D пространстве абсолютно инвариантна к повороту камеры.

*Новый код (добавить в `scoring.py`):*
```python
def calc_3d_perimeter(points_array):
    """Сумма 3D-расстояний между последовательными точками контура."""
    perimeter = 0.0
    for i in range(len(points_array)):
        p1 = points_array[i]
        p2 = points_array[(i + 1) % len(points_array)] # замыкаем контур
        perimeter += np.linalg.norm(p2 - p1)
    return perimeter

# В extract_macro_bone_metrics:
perimeter_L = calc_3d_perimeter(orbit_L_pts)
perimeter_R = calc_3d_perimeter(orbit_R_pts)

# Асимметрия периметров глазниц (1.0 = идеальная симметрия)
# Всегда делим меньшее на большее, чтобы диапазон был [0, 1]
metrics["orbital_perimeter_symmetry"] = min(perimeter_L, perimeter_R) / (max(perimeter_L, perimeter_R) + 1e-8)
```

#### ЗАМЕНА 4: `forehead_slope_index` ➔ `nasion_glabella_angle`
**Почему шумело:** Использовался угол нормали всего лба, что ломалось от мимики бровей и размера маски.
**Новая логика:** Считаем строгий профильный угол между Nasion (корень носа) и Glabella (межбровье) относительно вертикальной оси лица. Это жесткая костная структура, не зависящая от бровей.

*Новый код (добавить в `scoring.py`):*

```python
# В extract_macro_bone_metrics (нужно добавить Glabella в MACRO_BONE_INDICES):
```

Мы не удалили ни одного "шумного" признака. Вместо этого мы:

### 🌐 ЭТАП 2.1: СОЗДАНИЕ КАНОНИЧЕСКОГО ПРОСТРАНСТВА (`alignment.py`)

Нам необходимо написать функцию, которая доводит одиночный меш до «идеального» ракурса его бакета (например, ровно 0° для анфаса, ровно -45° для 3/4). Мы не будем использовать старую функцию `euler_to_rotation_matrix`, так как разработчик №8 доказал, что в ней перепутаны оси (баги A-01, CP-03). Мы перейдем на железобетонный стандарт `scipy`.

*Добавить в `backend/pipeline/alignment.py`:*

```python
import numpy as np
from scipy.spatial.transform import Rotation

# Словарик идеальных углов YAW для каждого бакета (уже есть в коде, дополняем)
_CANONICAL_YAW_BY_VIEW_GROUP = {
    "frontal": 0.0,
    "left_threequarter_light": -22.5, "right_threequarter_light": 22.5,
    "left_threequarter_mid": -45.0,   "right_threequarter_mid": 45.0,
    "left_threequarter_deep": -67.5,  "right_threequarter_deep": 67.5,
    "left_profile": -90.0,            "right_profile": 90.0,
}

def canonicalize_vertices_for_bucket(vertices: np.ndarray, angles_deg: np.ndarray, view_group: str) -> np.ndarray:
    """
    Приводит 3D-меш к канонической системе координат бакета.
    Нейтрализует наклон (pitch), крен (roll) и доводит поворот (yaw) до идеала.
    
    :param vertices: Сырые вершины из 3DDFA (N, 3)
    :param angles_deg: Углы из 3DDFA строго в формате [pitch, yaw, roll]
    :param view_group: Название бакета (например, 'frontal')
    """
    pitch, yaw, roll = angles_deg
    target_yaw = _CANONICAL_YAW_BY_VIEW_GROUP.get(view_group, 0.0)
    
    # 1. Строим матрицу ТЕКУЩЕГО поворота. 
    # Внимание: 3DDFA использует порядок вращения X(pitch), Y(yaw), Z(roll).
    # Знак минус используется для инверсии осей 3DDFA к стандартной правой СК.
    R_current = Rotation.from_euler('xyz', [-pitch, -yaw, -roll], degrees=True).as_matrix()
    
    # 2. Строим матрицу ЦЕЛЕВОГО поворота (Pitch и Roll жестко равны нулю!)
    R_target = Rotation.from_euler('xyz', [0.0, -target_yaw, 0.0], degrees=True).as_matrix()
    
    # 3. Вычисляем компенсирующую дельта-матрицу
    R_align = R_current.T @ R_target
    
    # 4. Вращаем меш строго вокруг его центроида, чтобы избежать смещения в космос
    centroid = vertices.mean(axis=0)
    aligned_vertices = (vertices - centroid) @ R_align + centroid
    
    return aligned_vertices
```

### 🎭 ЭТАП 2.3: РЕМОНТ ДЕТЕКТОРА МИМИКИ (`zones.py`)

Разработчик №8 нашел критический баг **ZN-02**: порог мимики умножался на `face_scale`. Из-за этого `jaw_excluded` всегда был `False`, и маска мимики не срабатывала НИКОГДА.

*Участок на замену в `backend/pipeline/zones.py` (функция `apply_expression_exclusion_mask`):*

```python
# БЫЛО:
# jaw_excluded = jaw_open_intensity > (THRESHOLD_JAW_OPEN * face_scale)
# smile_excluded = smile_intensity > (THRESHOLD_SMILE * face_scale)

# СТАЛО:
def apply_expression_exclusion_mask(base_mask, exp_params, bfm_indices):
    """
    BFM exp_params (коэффициенты базиса выражений) НЕ зависят от физического 
    масштаба сетки (face_scale). Они безразмерны. Умножать на scale — фатальная ошибка.
    """
    THRESHOLD_JAW_OPEN = 0.12
    THRESHOLD_SMILE = 0.8
    
    # 0-й параметр 3DDFA отвечает за открытие челюсти
    jaw_open_intensity = abs(exp_params[0]) 
    jaw_excluded = jaw_open_intensity > THRESHOLD_JAW_OPEN
    
    # 1-й и 2-й параметры отвечают за улыбку (растяжение губ)
    smile_intensity = max(abs(exp_params[1]), abs(exp_params[2]))
    smile_excluded = smile_intensity > THRESHOLD_SMILE

    excluded_flags = {
        "jaw_excluded": jaw_excluded,
        "smile_excluded": smile_excluded
    }
    
    # Мы не модифицируем саму 3D-маску, мы возвращаем флаги для жесткого 
    # удаления скомпрометированных метрик на этапе экстракции.
    return base_mask, excluded_flags
```

```python
import cv2
import numpy as np

def evaluate(self, img_path: str, bbox: dict = None) -> dict:
    """
    Оценивает качество фото. Защищает пайплайн от падений и отсеивает 
    нерелевантные (слишком мелкие или шумные) лица.
    """

    if img is None:
        # [FIX QG-01]: Вместо обрушения всего пайплайна возвращаем статус мягкого отказа
        return {
            "success": False,
            "is_rejected": True, 
            "reason": "INSUFFICIENT_DATA_UNREADABLE",
            "overall_score": 0.0
        }

    h, w = img.shape[:2]
    
    # [FIX QG-04]: Проверка минимального разрешения лица. 
    # Форензика текстуры невозможна, если лицо меньше 150x150 пикселей.
    if bbox is not None:
        face_h = bbox.get("h", h)
        if face_h < 150:
            return {
                "success": True,
                "is_rejected": True,
                "reason": f"FACE_TOO_SMALL_{int(face_h)}px",
                "overall_score": 0.1
            }

    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    
    # Оценка размытия (Лапласиан)
    blur_score = cv2.Laplacian(gray, cv2.CV_64F).var()
    
    # [FIX QG-02]: Порог резкости должен быть динамическим или более реалистичным.
    # Для соцсетей Laplacian variance > 65 уже считается приемлемым.
    sharpness_score = float(np.clip(blur_score / 150.0, 0.0, 1.0))
    
    # [FIX QG-03]: Оценка шума. Шум не должен уводить quality в минус.
    noise_score = self._estimate_noise(gray)
    noise_quality = float(np.clip(1.0 - (noise_score / 25.0), 0.0, 1.0))
    
    # Итоговый скор. Шумное, но резкое фото получит сбалансированную оценку
    overall_score = (sharpness_score * 0.7) + (noise_quality * 0.3)
    
    return {
        "success": True,
        "is_rejected": overall_score < 0.45, # Строгий форензик-порог
        "blur_value": blur_score,
        "noise_level": noise_score,
        "overall_score": overall_score
    }
```

### 🔬 ЭТАП 3.3: ВОСКРЕШЕНИЕ ТЕКСТУРНОЙ МАТЕМАТИКИ (`texture.py`)

*Участок на замену в `backend/pipeline/texture.py`:*

**1. Фикс детектора студийного света (Сломанная формула медианы)**
Освещение лица никогда не давало результат, потому что `(0.35 - shadow_area)` всегда было отрицательным (медиана всегда делит пиксели 50/50).

```python
# СТАЛО (внутри SkinTextureAnalyzer._detect_studio_lighting):
def _detect_studio_lighting(self, gray: np.ndarray, face_mask: np.ndarray) -> float:
    face_pixels = gray[face_mask > 0]
    if len(face_pixels) == 0: return 0.0
    
    # Студийный свет (софтбоксы) убивает жесткие тени, делая дисперсию освещения минимальной.
    # Естественный свет (солнце, лампа на потолке) дает жесткий градиент.
    std_illumination = float(np.std(face_pixels))
    
    # Нормализуем. Чем ниже std (равномернее свет), тем выше вероятность студии/вспышки.
    # Если std < 30 (очень плоский свет) -> score 1.0. Если std > 70 (жесткие тени) -> score 0.0.
    uniformity_score = np.clip(1.0 - ((std_illumination - 30.0) / 40.0), 0.0, 1.0)
    
    # Блики от вспышки (пересветы)
    highlights = np.sum(face_pixels > 240) / len(face_pixels)
    highlight_score = np.clip(highlights * 10.0, 0.0, 1.0)
    
    return float(np.clip((uniformity_score * 0.7) + (highlight_score * 0.3), 0.0, 1.0))
```

**2. Фикс детектора бликов кожи (`specular_gloss`)**
Брал 90-й перцентиль яркости всего лица. В него всегда попадали белки глаз и зубы из-за улыбок. 

```python
# СТАЛО (внутри SkinTextureAnalyzer):
def _compute_specular_gloss(self, hsv_img: np.ndarray, skin_mask: np.ndarray) -> float:
    # [FIX TX-01]: Извлекаем канал Яркости (V)
    v_channel = hsv_img[:, :, 2]
    
    # КРИТИЧНО: Эродируем маску сильно (на 10-15 пикселей), чтобы гарантированно
    # исключить глаза, зубы и блики на краях губ. Оставляем только щеки, лоб, нос.
    kernel = np.ones((11, 11), np.uint8)
    strict_skin_mask = cv2.erode(skin_mask, kernel, iterations=2)
    
    safe_pixels = v_channel[strict_skin_mask > 0]
    if len(safe_pixels) < 100: return 0.0
    
    # Считаем 95-й перцентиль (самые яркие участки КОЖИ, а не зубов)
    gloss_value = float(np.percentile(safe_pixels, 95))
    
    # Нормируем. Глянец обычно лежит в диапазоне 180-255.
    return float(np.clip((gloss_value - 150) / 105.0, 0.0, 1.0))
```

**3. Фикс плотности пор (`pore_density`)**
Функция Лапласиана (`cv2.Laplacian` с типом `CV_64F`) выдавала огромные значения дисперсии, которые делились на `255.0` вместо возведения в квадрат, что превращало метрику в неадекватные числа (CV 48%+).

```python
# СТАЛО (внутри SkinTextureAnalyzer):
def _compute_pore_density(self, gray: np.ndarray, skin_mask: np.ndarray) -> float:
    # Оставляем только кожу
    masked_gray = cv2.bitwise_and(gray, gray, mask=skin_mask)
    
    # Вычисляем Лапласиан (выделение высокочастотных деталей - пор, мелких текстур)
    laplacian = cv2.Laplacian(masked_gray, cv2.CV_64F)
    
    # Считаем дисперсию только на валидных пикселях маски
    valid_laplacian = laplacian[skin_mask > 0]
    if len(valid_laplacian) == 0: return 0.0
    
    variance = np.var(valid_laplacian)
    
    # [FIX TX-05]: Правильная нормализация для CV_64F. 
    # Дисперсия Лапласиана для кожи обычно лежит в пределах 100 - 1500.
    normalized_density = float(np.clip(variance / 1000.0, 0.0, 1.0))
    
    return normalized_density
```

### 🛑 ЭТАП 4.1: ФИКС ПАРНОГО ВЫРАВНИВАНИЯ (`compare.py`)

**Суть проблемы:** Алгоритм `rigid_umeyama` вызывался с параметром `allow_scale=True` (баг A-03). Алгоритм буквально «сдувал» или «раздувал» череп из 2025 года, чтобы он совпал с черепом из 1999 года. Это преступление в судебной экспертизе: масштаб костей у взрослого человека не меняется. Кроме того, алгоритм не получал косинусные веса видимости, из-за чего скошенные грани профиля ломали матрицу выравнивания (баг E-25).

*Участок на замену (в `backend/pipeline/compare.py` внутри `align_and_score`):*

```python
# БЫЛО:
# alignment = rigid_umeyama(points_a_unit[shared_idx], points_b_unit[shared_idx], allow_scale=True)

# СТАЛО:
def align_and_score(recon_a, recon_b, shared_idx, vis_a, vis_b):
    # 1. Извлекаем общие видимые точки (уже в каноническом пространстве после Итер. 2!)
    pts_a = recon_a.vertices_canonical[shared_idx]
    pts_b = recon_b.vertices_canonical[shared_idx]
    
    # 2. Формируем строгие веса. Берем минимальный вес видимости между двумя фото.
    # Если вершина на краю лица на фото А (вес 0.1), а на фото В по центру (вес 0.9), 
    # ее итоговый вес = 0.1. Мы не доверяем скошенным зонам.
    combined_weights = np.minimum(
        vis_a.cosine_weights[shared_idx], 
        vis_b.cosine_weights[shared_idx]
    )
    
    # 3. ФОРЕНЗИК-ВЫРАВНИВАНИЕ (Rigid Umeyama)
    alignment = rigid_umeyama(
        pts_a, 
        pts_b, 
        allow_scale=False,  # [AXIOM-02 STRICT ENFORCEMENT]: Масштаб заблокирован!
        weights=combined_weights
    )
    
    # 4. Вычисляем физическую ошибку (RMS) только по хорошо видимым точкам
    # Игнорируем 10% самых больших выбросов (trimmed mean), чтобы волосы/артефакты не ломали скор
    errors = np.linalg.norm(alignment.source_aligned - pts_b, axis=1)
    k_keep = int(len(errors) * 0.9)
    # Используем partition (O(N)), а не sort (O(N log N)) для скорости
    cutoff_error = np.partition(errors, k_keep - 1)[k_keep - 1]
    
    # Считаем взвешенную ошибку
    valid_mask = errors <= cutoff_error
    robust_error = np.average(errors[valid_mask], weights=combined_weights[valid_mask])
    
    return alignment, robust_error
```

```python
def decompose(self, raw_zone_errors: dict, nuisance: NuisanceDeltas, bucket: str) -> CalibratedComparisonResult:
    """
    Разделяет сырую ошибку на СИГНАЛ (реальная разница) и ШУМ (ожидаемая погрешность),
    учитывая конкретную зону, ракурс и штрафы за мимику/качество.
    """
    zone_snr = {}
    signal_components = {}
    total_signal = 0.0
    total_noise = 0.0
    
    for zone, raw_error in raw_zone_errors.items():
        # 1. Получаем базовый шум строго для ЭТОЙ зоны в ЭТОМ бакете
        profile = self.model.zone_profiles.get(f"{bucket}_{zone}")
        if not profile:
            continue
            
        # 2. Вычисляем динамический ожидаемый шум (Dynamic MAD)
        # Если качество упало или поза сильно отличается, ожидаемый шум растет.
        expected_noise = profile.base_mad + (nuisance.pose_delta_mag * profile.pose_weight) + \
                         (nuisance.quality_degradation * profile.quality_weight)
                         
        # 3. Декомпозиция
        # Сигнал — это то, что осталось после вычитания ожидаемого шума.
        signal = max(0.0, raw_error - expected_noise)
        
        # 4. Расчет SNR (Signal-to-Noise Ratio)
        snr = signal / (expected_noise + 1e-6)
        
        zone_snr[zone] = snr
        signal_components[zone] = signal
        
        total_signal += signal
        total_noise += expected_noise

    # Итоговый глобальный SNR для пары
    final_snr = total_signal / (total_noise + 1e-6)
    
    return CalibratedComparisonResult(
        final_snr=final_snr,
        is_significant_anomaly=(final_snr > 2.5), # Консервативный порог 2.5 Сигмы
        zone_snr=zone_snr
    )
```

```python
def analyze_timeline_calibrated(
    timeline_photos: List[dict], 
    pair_engine: PairComparisonEngine, 
    reference_date: str = "2001-12-31"
) -> TimelineReport:
    """
    Проходит по временной шкале и оценивает откалиброванные SNR-скачки.
    """
    # Сортировка строго по datetime, а не строкам (Фикс бага CH-02)
    photos = sorted(timeline_photos, key=lambda x: datetime.fromisoformat(x['date']))
    
    baseline_photos = [p for p in photos if p['date'] <= reference_date]
    if not baseline_photos:
        baseline_photos = [photos[0]]
    primary_baseline = baseline_photos[0] # Можно заменить на медиану
    
    flags = []
    
    for i in range(1, len(photos)):
        prev_photo = photos[i-1]
        curr_photo = photos[i]
        
        # 1. Сравниваем с предыдущим годом (T_i vs T_{i-1})
        res_to_prev = pair_engine.compare(prev_photo, curr_photo)
        
        # 2. Сравниваем с эталоном молодости (T_i vs 1999)
        res_to_baseline = pair_engine.compare(primary_baseline, curr_photo)
        
        # 3. Логика Impossible Transition (Резкая подмена)
        # Если SNR > 3.0 (сигнал превышает шум в 3 раза), и это не объясняется мимикой
        if res_to_prev.final_snr > 3.0:
            flags.append({
                "type": "IMPOSSIBLE_TRANSITION",
                "date": curr_photo['date'],
                "snr": res_to_prev.final_snr,
                "anomalous_zones": [z for z, snr in res_to_prev.zone_snr.items() if snr > 3.0]
            })
            
        # 4. Логика Return To Reference (RTR)
        # Если человек резко изменился по сравнению с прошлым годом (SNR_prev > 2.5), 
        # НО при этом его лицо идеально совпадает с лицом 1999 года (SNR_baseline < 1.0)
        if res_to_prev.final_snr > 2.5 and res_to_baseline.final_snr < 1.5:
             flags.append({
                "type": "RETURN_TO_REFERENCE",
                "date": curr_photo['date'],
                "evidence": f"Matched baseline {primary_baseline['date']} despite breaking continuity from {prev_photo['date']}"
            })

    return TimelineReport(flags=flags)
```

### 🧠 ЭТАП 4.4: ЛОГИРОВАНИЕ И АПОСТЕРИОРНЫЕ ВЕРОЯТНОСТИ (`verdict.py`)

Наконец, мы исправляем баги Байесовского перемножения (I-37, C-16, I-38). Мы не перемножаем вероятности «в лоб» (что дает `1.6e-4` и ломает float-лимиты). Мы считаем всё в безопасном `log-домене` и возвращаем правильные Enum-статусы.

*Участок на замену (в `backend/pipeline/verdict.py`):*

```python
def bayesian_update_log_domain(priors: dict, likelihoods: dict) -> dict:
    """
    Безопасное обновление вероятностей (H0: Same, H1: Fake, H2: Different).
    """
    # 1. Переводим в логарифмы для защиты от Underflow (Баг I-37)
    log_posteriors = {}
    for hyp in ["H0", "H1", "H2"]:
        log_prior = np.log(priors.get(hyp, 0.33) + 1e-9)
        log_like = np.log(likelihoods.get(hyp, 0.33) + 1e-9)
        log_posteriors[hyp] = log_prior + log_like
        
    # 2. Безопасная экспонента (Log-Sum-Exp trick)
    max_log = max(log_posteriors.values())
    posteriors = {k: np.exp(v - max_log) for k, v in log_posteriors.items()}
    
    # 3. Нормализация (сумма = 1.0)
    total = sum(posteriors.values())
    final_probs = {k: v / total for k, v in posteriors.items()}
    
    return final_probs
```

### Итог Итерации 4
Выполнив этот код, система перестанет гадать по сырым векторам. Теперь конвейер работает так:
1. Берем два фото.
2. Идеально выравниваем их черепа, запрещая изменение масштаба.
3. Считаем ошибку в миллиметрах (RMS).
4. Вычисляем динамический шум (насколько эти фото мыльные или скошенные).
5. Считаем SNR (Сигнал / Шум).
6. Если SNR > 3.0 — Байес фиксирует аномалию.

Это и есть финальная точка проекта. Математика, геометрия, текстуры и статистика слиты воедино.
